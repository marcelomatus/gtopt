# `ieee_118b` — IEEE 118-Bus Test Case

The IEEE 118-bus test case represents a portion of the American Electric Power
System (in the Midwestern US) as it was in the early 1960s.  It is the most
widely used medium-scale benchmark in generation and transmission expansion
planning (GTEP) literature.

## Source

- **Original data**: Iraj Dabbagchi (AEP), converted to IEEE Common Data Format
  by Rich Christie (University of Washington) in August 1993.
- **pglib-opf**: Standardised OPF benchmark with line limits and cost data at
  https://github.com/power-grid-lib/pglib-opf
- **Pandapower**: `pandapower.networks.case118()` (MATPOWER conversion).

## System data

| Quantity | Value |
|----------|-------|
| Buses | 118 |
| Generators | 54 |
| Loads | 99 |
| Branches | 186 (173 lines + 13 transformers) |
| Base voltage | 138 kV / 345 kV |
| Base MVA | 100 |

## Generating golden files

Pandapower's built-in `case118` network lacks active-power cost data, so DC OPF
does not converge out of the box.  Golden files should be generated by running
pandapower DC OPF on a cost-augmented network:

```python
import pandapower as pp
import pandapower.networks as pn
import numpy as np

net = pn.case118()

# Add linear generation costs (cp1) — see pglib-opf for reference values
np.random.seed(0)
for i in net.gen.index:
    net.poly_cost.loc[len(net.poly_cost)] = {
        "element": i, "et": "gen",
        "cp0_eur": 0.0, "cp1_eur_per_mw": np.random.uniform(10, 50),
        "cp2_eur_per_mw2": 0.0,
        "cq0_eur": 0.0, "cq1_eur_per_mvar": 0.0, "cq2_eur_per_mvar2": 0.0,
    }

# Alternatively: convert from pglib-opf MATPOWER file
# python -m pp2gtopt pglib_opf_case118_ieee.m -o cases/ieee_118b/ieee_118b.json

pp.rundcopp(net)
# Write net.res_* to cases/ieee_118b/output/ as gtopt CSV files
```
