# Integration Tests

This directory contains end-to-end (e2e) integration tests for the **gtopt**
solver.  Each test case runs the gtopt standalone binary on a power system
JSON file, validates the solution, and compares the output CSV files against
golden reference files.

## Test structure

Every case under `cases/<name>/` contains:

```
cases/<name>/
  <name>.json          — system definition (buses, generators, lines, …)
  output/              — golden reference CSV files
  output_<solver>/     — solver-specific golden files (optional)
  planning.json        — options file (optional)
```

A case is registered in `integration_test/CMakeLists.txt` with:

```cmake
add_e2e_case(<name> <json_file>)
```

which creates three CTest tests:

| Test name | What it checks |
|-----------|----------------|
| `e2e_<name>_solve` | Runs `gtopt <json> --output-directory <dir>` |
| `e2e_<name>_validate_solution` | Checks `solution.csv` exists with `status=0` and all expected output files are present |
| `e2e_<name>_compare_<csv>` | Compares one CSV output file against the golden reference (one test per file) |

## Golden files

Golden reference files are stored under `cases/<name>/output/`.  Each golden
file is a flat CSV matching the format that gtopt produces after reassembling
per-(scene, phase) shards.  The `run_gtopt.cmake` script handles shard
reassembly transparently.

When `GTOPT_SOLVER` is set and a matching `cases/<name>/output_<solver>/`
directory exists, the comparison uses `--strict` mode (exact numeric match).
Otherwise the generic `output/` golden files are used with tolerance `1e-6`.

### How golden files are generated

Golden files are generated by running the **pandapower** DC OPF solver on the
same system, then converting the result into gtopt's CSV output format.
Pandapower is an independent power system analysis tool that does not share
code with gtopt, so it serves as an unbiased reference.

The workflow is:

1. Convert the gtopt JSON to a pandapower network using
   `scripts/gtopt2pp/convert.py`
2. Run `pandapower.rundcopp(net)` to solve the DC optimal power flow
3. Extract solution values (bus angles, generator dispatch, line flows,
   demand served, duals) from the pandapower result dataframes
4. Write them in gtopt's CSV output format under `cases/<name>/output/`

This process is implemented in `tools/generate_golden.py` (see below).

### Solver-specific golden files

Some test cases have numerically degenerate LP solutions where different
solvers produce equally-valid but different primal results.  For these cases:

- `output_cplex/`, `output_highs/`, `output_mindopt/` — solver-specific
  golden directories
- The `compare_csv.cmake` script checks `$GTOPT_SOLVER` and prefers the
  solver-specific directory when it exists
- Comparisons in solver-specific mode use `--strict` (exact match)
- Comparisons in generic mode use `--tolerance 1e-6` (non-strict)

### Regenerating golden files

To regenerate golden files for all cases after a formulation change:

```bash
# Ensure pandapower is installed
pip install pandapower

# Run the golden file generator
python tools/generate_golden.py --cases-dir cases/
```

Or for a single case:

```bash
python tools/generate_golden.py --cases-dir cases/ --case ieee_118b
```

The generator reads each system JSON, converts to pandapower, solves the
DC OPF, and writes the output files.

## Adding a new test case

1. Place the system JSON under `cases/<name>/<name>.json`
2. Generate golden files:
   ```bash
   python tools/generate_golden.py --cases-dir cases/ --case <name>
   ```
3. Register in `integration_test/CMakeLists.txt`:
   ```cmake
   add_e2e_case(<name> <name>.json)
   ```
4. Build and test:
   ```bash
   cmake --build build -j$(nproc)
   ctest --output-on-failure -R "e2e_<name>"
   ```

## IEEE test case sources

Standard IEEE test cases are widely available in MATPOWER format:

- **IEEE 4–118 bus**: University of Washington Power Systems Test Case Archive
  (https://www.ee.washington.edu/research/pstca/)
- **IEEE 24-bus RTS**: IEEE Reliability Test System
- **pglib-opf**: Standardized OPF benchmarks with line limits and cost data
  (https://github.com/power-grid-lib/pglib-opf)

Chinese provincial grid data (e.g. 3266-bus system) is available from IEEE
DataPort (DOI: 10.21227/vma9-wk20).
