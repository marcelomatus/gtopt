# plexos-coad fixtures

PLEXOS XML test cases vendored from
[Computational-Energy/plexos-coad](https://github.com/Computational-Energy/plexos-coad)
(BSD-3-Clause, NREL 2018 — see `LICENSE`). These are used as
**reader-portability** fixtures: they exercise the
`plexos2gtopt.plexos_xml` MasterDataSet parser against schemas
generated by an independent PLEXOS workflow (different class IDs,
different property catalogues, no companion CSV time-series).

| File | Source path in plexos-coad | Buses | Generators | Lines | Compressed size |
|---|---|---|---|---|---|
| `118-Bus.xml.xz` | `coad/test/118-Bus.xml` | 118 | 54 | 186 | ~80 KB |
| `RTS-96.xml.xz` | `coad/test/RTS-96.xml` | 73 | 99 | 120 | ~109 KB |

Both files are LZMA-compressed in-place; the test reader decompresses
into a tempdir on demand. The original `coad/test/` directory also
ships much larger fixtures (`Solar33P.xml` 34 MB, `WFIP-MISO.xml`
81 MB) which are out of scope for the unit-test suite.

## Why not full-LP integration tests yet?

The CEN PCP daily bundle pairs `DBSEN_PRGDIARIO.xml` with per-class
CSV schedules (`Gen_Rating.csv`, `Nod_Load.csv`, …) that carry the
per-hour dispatch envelope. These plexos-coad cases ship the XML
database alone — Pmax/Pmin/Load live inside the XML `t_data` table.
Adding a t_data fallback to the per-class extractors is on the P3
roadmap; once that lands, `118-Bus.xml.xz` becomes a full `gtopt
--lp-only` integration fixture.

## Regenerating from source

```bash
curl -L -O https://raw.githubusercontent.com/Computational-Energy/plexos-coad/master/coad/test/118-Bus.xml
curl -L -O https://raw.githubusercontent.com/Computational-Energy/plexos-coad/master/coad/test/RTS-96.xml
xz -9 118-Bus.xml RTS-96.xml
```
